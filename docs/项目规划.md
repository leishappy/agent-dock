# AgentDock — 项目整体规划

## 一句话

**AI 开发者桌面伴侣**：带 USB Hub 的 ESP32 桌面小屏幕，专门显示 AI Agent 任务进度。

---

## 解决了什么问题

AI Agent（Claude Code、Cursor、Copilot 等）的运行进度都在 IDE/终端里。切来切去看进度很烦。

AgentDock 是一块独立小屏幕，**看一眼就知道 Agent 在干嘛**：任务进度、是否出错、需要介入。

---

## 核心功能

### V1.0 MVP

- [x] **3 口 USB Hub**：USB-C 一线连 PC，分出 3 个 USB-A
- [x] **屏幕显示 Agent 状态**：任务进度条、完成/失败数、当前任务名
- [x] **USB CDC 通信**：PC 端通过虚拟串口发数据给屏幕
- [x] **独立联网**：WiFi 连接后，电脑关机也能显示天气、时钟
- [x] **BLE 手机通知**：手机通知推送到屏幕

### V2.0 扩展

- [ ] **通知提醒**：Agent 报错时屏幕闪烁/蜂鸣
- [ ] **自定义面板**：用户可配置显示内容
- [ ] **多 Agent 支持**：同时监控 Claude Code + Cursor
- [ ] **历史统计**：今日用了多少 token、完成任务数

## 硬件架构

```
USB-C (PC)
  │
  ├── GL850G USB Hub
  │     ├── USB-A 口 1 (接键盘/鼠标/U盘)
  │     ├── USB-A 口 2
  │     └── USB-A 口 3
  │
  ├── ESP32-S3-WROOM-1
  │     ├── USB CDC ← PC 发 JSON 数据
  │     ├── WiFi → 独立联网（天气/时钟/OTA）
  │     ├── BLE → 手机通知推送
  │     └── SPI → ST7789 240×240 屏幕
  │
  └── 供电：USB-C 5V（整板供电）
```

## 软件架构

### 数据流

```
┌─────────────────┐     USB CDC      ┌──────────────┐
│  PC (Python)    │ ───────────────→ │  ESP32-S3    │
│                 │   JSON @ 115200  │              │
│  agent_monitor  │                  │  解析 JSON   │
│    ↓            │                  │    ↓         │
│  监听 Agent 日志 │                  │  刷新屏幕    │
│    ↓            │                  │              │
│  采集任务进度    │                  │  LVGL 渲染   │
└─────────────────┘                  └──────────────┘
```

### PC 端 (Python)

| 组件 | 功能 |
|------|------|
| `agent_monitor.py` | 监听 Claude Code / Cursor 日志，解析任务进度 |
| `serial_sender.py` | 格式化 JSON 通过串口发送 |
| `tray_app.py` | 系统托盘图标，后台运行 |

### ESP32 固件 (ESP-IDF)

| 组件 | 功能 |
|------|------|
| `usb_cdc` | USB CDC 接收数据，解析 JSON |
| `display` | ST7789 SPI 驱动 |
| `ui` | LVGL 渲染界面（进度条、图标、文字） |
| `wifi_app` | WiFi 连接，NTP 时钟，天气 API |
| `ble_notify` | BLE GATT 接收手机通知 |
